Hanya's File Structure

To Generate a HTML Page, Hanya needs at least one Page and one Template file. To keep this files clean divide your pages into reuseable partials and blocks. Partials can loaded by a "include" tag and will be placed immediately at his space. The Blocksystem allows you to attach Blocks and HTML Code to an id, which can be rendered in a template or other file.

File Structure

Images, Stylesheets and Javascript files are stored in the "public" directory. This is the only directory that is accessible from outside. Includeable partials and attachable blocks are stored in the "elements" directory. They are marked with an underscore for partial and a plus for block.

Templates

<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>{hanya:read(title|Default Title|)}</title>
    <link rel="stylesheet" href="{hanya:file(reset.css)}" type="text/css">
    <link rel="stylesheet" href="{hanya:file(style.css)}" type="text/css" media="screen">
    <script src="{hanya:file(jquery.js)}" type="text/javascript"></script>
    <script src="{hanya:file(hanya.js)}" type="text/javascript"></script>
  </head>
  <body>
    {hanya:toolbar()}
    <header>
      {hanya:include(header)}
    </header>
    <div id="content">
      {hanya:block(content)}
      {hanya:block(sidebar)}
    </div>
    <footer>
      {hanya:benchmark()}
    </footer>
  </body>
</html>

Partial & Block

<h1>Site Title</h2>
<p>Sub Title or Slogan</p>
<ul>
  <li><a href="{hanya:link()}">Index</a></li>
  <li><a href="{hanya:link(page)}">Page</a></li>
  <li><a href="{hanya:link(directory/subpage)}">Sub Page</a></li>
</ul>
